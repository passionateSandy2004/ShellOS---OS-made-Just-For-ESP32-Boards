Platform

Six integrated layers.
One complete platform.

ShellOS is not just a firmware. It is a full operating platform covering shell access, package lifecycle, persistent storage, networking, a build toolchain, and package distribution — all working together seamlessly.

💻

Interactive Shell

A full command-driven workflow for diagnostics, runtime visibility, task inspection, and network operations — accessible over USB serial or TCP.

📦

Package Lifecycle

Install, start, stop, and remove software packages on the device at runtime — with no firmware reflash required. Operations take seconds, not minutes.

💾

Persistent Storage

LittleFS-backed filesystem at /root with isolated per-package data and log directories that survive power cycles.

📡

Remote Access

TCP shell server on port 2323 for remote command sessions and a full HTTP REST API on port 8080 for programmatic package lifecycle control.

🔧

Package Toolchain

Transpile Arduino-style .ino sketches to Lua and compile .shpkg packages directly from the desktop Imager in one click.

🔗

App Store & Distribution

Publish packages to GitHub with one click, share a repository link, and let anyone import with automatic SHA-256 integrity verification before install.

Architecture

Built for operational workflows,
not just demo sketches.

Every layer is purpose-built so the system stays stable under real continuous-operation conditions — persistent state, live service management, and multi-package environments.

Layer 01

Kernel & Runtime

Lua-based task scheduler on top of ESP-IDF. Packages run as isolated Lua tasks managed by the core runtime.

Layer 02

Filesystem

LittleFS provides wear-leveled, crash-resilient storage. Each package gets its own directory at /root/packages/<name>.

Layer 03

Network Stack

Dual interface: TCP shell server for interactive remote sessions and an HTTP REST server for API-driven automation.

Layer 04

Package Format

The .shpkg binary bundles compiled Lua, manifest JSON, and assets in a single portable archive with embedded metadata.

Layer 05

Toolchain

Arduino-to-Lua transpiler converts familiar .ino syntax so existing embedded developers have a zero-learning-curve onramp.

Layer 06

Distribution

Git-based publish flow with shellos-package.json manifests and SHA-256 checksums for verifiable, trustworthy package sharing.

Windows Imager

From zero to running in minutes.

The ShellOS Imager bundles every tool you need — firmware flashing, serial monitoring, package building, deployment, and the App Store — into one Windows desktop application.

Track 01

Flash & Connect

1
Open ShellOS Imager. Select your board target (ESP32-CAM or ESP32-C6) and choose the USB serial port.
2
Click Flash. The Imager writes the bootloader, partition table, and full OS image in one operation.
3
Use the serial monitor at 115200 baud for first-boot WiFi setup. Board IP is shown on-screen once connected.
4
Switch to the Packages tab, enter the board IP, and connect to load the live installed package inventory.
Track 02

Build, Deploy & Share

1
Select an Arduino-style .ino sketch. Click Build to transpile and compile into a .shpkg package.
2
Click Upload & Install. The package is deployed to the board over WiFi and appears in the installed packages table immediately.
3
Use the table to Run, Stop, or Remove any installed package without touching the base firmware.
4
Hit Publish to push the package to your GitHub repo. Share the link — anyone can import and verify it instantly.
